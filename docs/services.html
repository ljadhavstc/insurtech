Services

API services and mock server

API Service

Location: src/services/api.ts

Axios instance with interceptors for authentication and token refresh.

Features

  • Automatic auth token attachment from Zustand store
  • 401 error handling with automatic token refresh
  • Request/response interceptors
  • Base URL configuration
  • Timeout handling

Usage

import api from '@/services/api';

// GET request
const response = await api.get('/users');

// POST request
const response = await api.post('/auth/login', {
  email: 'user@example.com',
  password: 'password123',
});

// PUT request
const response = await api.put('/users/1', {
  name: 'John Doe',
});

// DELETE request
const response = await api.delete('/users/1');

Request Interceptor

Automatically attaches Authorization: Bearer <token> header from auth store.

Response Interceptor

On 401 Unauthorized:

  1. Attempts to refresh token using refresh token
  2. Retries original request with new token
  3. Queues concurrent requests during refresh
  4. Logs out user if refresh fails

Mock API endpoints for development with realistic responses and latency simulation.

OTP codes are logged to console in development mode.

Latency Simulation

All mock endpoints simulate network latency:

  • Login: ~1000ms
  • Register: ~1200ms
  • Forgot Password: ~800ms
  • OTP Verify: ~600ms
  • Reset Password: ~800ms
  • Refresh Token: ~500ms
