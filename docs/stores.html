Auth Store

Location: src/stores/authStore.ts

Zustand store for authentication state management with AsyncStorage persistence.

State

PropertyTypeDescription
userUser | nullCurrent user object
tokenstring | nullAccess token
refreshTokenstring | nullRefresh token
isAuthenticatedbooleanAuthentication status

Actions

ActionParametersDescription
setTokenstoken: string, refreshToken: stringSet auth tokens
setUseruser: UserSet current user
loginuser: User, token: string, refreshToken: stringComplete login flow
logout-Clear auth state
updateUseruser: Partial<User>Update user data

Usage

import { authStore } from '@/stores/authStore';

// Get state
const user = authStore((state) => state.user);
const isAuthenticated = authStore((state) => state.isAuthenticated);
const token = authStore((state) => state.token);

// Get actions
const login = authStore((state) => state.login);
const logout = authStore((state) => state.logout);
const setTokens = authStore((state) => state.setTokens);

// Use actions
login(user, token, refreshToken);
logout();
setTokens(newToken, newRefreshToken);

// Get multiple values
const { user, token, login, logout } = authStore();

Persistence

The store is persisted to AsyncStorage with the key auth-storage.

Persisted state includes:

  • user
  • token
  • refreshToken
  • isAuthenticated

State is automatically restored on app restart.
